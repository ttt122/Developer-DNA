# DevDNA

DevDNA analyzes public GitHub activity and turns it into a shared developer identity language.

It is not a personality test, and it is not only a README decoration tool.

DevDNA exists to make developer styles understandable through GitHub-native data.

```text
GitHub Activity
-> Developer Genome
-> Developer DNA Type
-> README Card / Timeline / Compatibility
```

## Core Principle

DevDNA starts from measurable behavior, not from type names.

The project order is:

1. Philosophy
2. Genome design
3. GitHub data mapping
4. Type design
5. Type dictionary
6. README card design
7. MVP implementation
